2. 관련 페이지

	1) makelist.jsp	: 회원만 작성 가능
		
		a. 영화 추가 (AJAX 사용)
			- AJAX 및 네이버 API 사용
			- 검색 : 영화제목
			- select 항목 : 포스터, 제목, 제작년도
			- 검색버튼 or 글자입력 하면 API에서 정보 받아와서 화면에 뿌리기 
			
		b. 영화 삭제 (jQuery 사용)
			- 삭제 버튼 클릭시 이미지 태그 삭제
		
		b. 유효성 테스트
			- 제목 입력
			- 내용 입력
			- 영화는 2개 이상 선택
			
		c. DB INSERT
			- 글번호, 회원 아이디, 게시판 코드, 제목, 작성일, 내용, 별점, 영화제목, 장르, 추천, 비추천, 신고, 공개여부, 조회수
				- Null : 별점, 장르, 
				- 0 : 추천, 비추천, 신고, 조회수
				- 1 : 공개여부
				- 2 : 게시판 코드
				- TimeStamp : 작성일
				- 영화제목은 |로 연결
				- 글번호는 시퀀스 사용 
				- 회원 아이디는 세션에서 가져올 것
			- 예외처리
			 
			
		
		
			
	2) listdetail.jsp
	
		a. 기본화면
		
			- List 상세내용 >> 이전페이지에서 글번호 받아야 함
				- select 항목 : 글번호, 회원ID, 제목, 작성일, 내용, 영화제목, 추천, 비추천, 조회수, 회원사진
				- join : 회원아이디를 통해 회원테이블에서 회원사진 가져올것
				- select 조건 : 글번호
				- 글번호는 파라미터로 갖다 쓰고, 결과페이지로 이동할 때 request의 attribute로 보낼 것
				- 수정/삭제/신고
					- 작성자-수정/삭제   |    外 - 신고
						- 작성자 여부는 세션의 회원아이디와 작성자 대조
					- 수정
						- makelist 페이지로 이동
						- select 항목 : 제목, 내용,영화제목
						- select 조건 : 글번호
						- insert 처리는 1번대로
					- 삭제
						- 삭제 여부 확인창 >> 확인/취소 묻는 팝업창 사용법 확인
						- delete 조건 : 글번호
					- 신고
						- 신고 여부 확인창 >> 확인/취소 묻는 팝업창 사용법 확인
						- update : 글번호로 찾아서 신고수 ++
							>> query문 고민해볼것
					
			- 댓글
				- 조회
					- select 항목 : 댓글 작성자 프로필 사진, 댓글 작성자, 댓글 작성일, 댓글 내용
					- select 조건 : 글번호로 join, 작성일 순서로 정렬
				- 작성
					- 한글 기준 500자 이하 (varchar2 1000임)
				- 수정/삭제/신고
				
		b. 페이지 이동
			- 이전 버튼 : 이전 페이지로 이동 >> 브라우저의 뒤로가기 버튼 활용 가능?
			- 유저 정보 : List/댓글 작성자의 상세페이지 이동 >> 회원 아이디 보내줘야 할 듯?
			- 영화 포스터 : 해당 영화 상세페이지로 이동  >> 영화 제목 보내주면 되나?
			
		c. 기능
			- List
				- 작성자
					- 수정/삭제
				- 外
					- 신고 
			- 댓글
				- 작성자
					- 작성
					- 수정/삭제
				- 外
					- 신고
